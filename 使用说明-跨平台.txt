Codex API Switcher 跨平台版使用说明
===================================

下载地址
--------

GitHub Release：
https://github.com/jiali9974-ai/CodexApiSwitcher/releases/tag/v2.1.0

macOS：
- Apple Silicon / M 系列：CodexApiSwitcher-macos-arm64.zip
- Intel Mac：CodexApiSwitcher-macos-x64.zip

Windows：
- CodexApiSwitcher-win-x64.exe

Linux：
- CodexApiSwitcher-linux-x64
- CodexApiSwitcher-linux-arm64

说明：macOS 的 .app 不直接放进 Git 仓库，因为 .app 内自包含主程序超过 GitHub 普通 Git 单文件 100 MB 限制。请从 Release 下载 zip，解压后就是可启动的 .app。


macOS 怎么打开
--------------

1. 下载对应架构的 zip：
   - M 系列芯片选 arm64。
   - Intel Mac 选 x64。
2. 在 macOS 上解压 zip。
3. 双击解压出来的 .app。
4. 如果系统提示来自未验证开发者：
   - 打开“系统设置”。
   - 进入“隐私与安全性”。
   - 找到 CAS 的拦截提示。
   - 点击“仍要打开”。
5. 如果快捷键或鼠标侧键没有反应：
   - 打开“系统设置”。
   - 进入“隐私与安全性 -> 辅助功能”。
   - 给 Codex API Switcher 授权。
6. 如果双击仍失败，在终端查看日志：
   tail -n 80 ~/Library/Logs/CodexApiSwitcher.log
7. 如需手动放行隔离属性：
   xattr -dr com.apple.quarantine ./CodexApiSwitcher-macos-arm64.app
   或：
   xattr -dr com.apple.quarantine ./CodexApiSwitcher-macos-x64.app

当前 macOS 包为 ad-hoc 签名，不是 Apple Developer ID 公证包，首次运行可能仍需要手动允许。


Windows 怎么打开
----------------

1. 双击 CodexApiSwitcher-win-x64.exe。
2. 选择包含 config.toml 的 Codex 根目录。
3. 如果 Windows 安全提示，请确认文件来源后再允许运行。


Linux 怎么打开
--------------

1. 先给文件加执行权限：
   chmod +x ./CodexApiSwitcher-linux-x64
   或：
   chmod +x ./CodexApiSwitcher-linux-arm64

2. 运行：
   ./CodexApiSwitcher-linux-x64
   或：
   ./CodexApiSwitcher-linux-arm64

Linux 桌面需要基础系统库，例如 glibc、libstdc++、ICU、fontconfig、X11/AppIndicator 相关库。全局快捷键和鼠标侧键以 X11 支持最好，Wayland 下取决于桌面环境权限策略。


第一次使用
----------

1. 先彻底退出 Codex。
2. 打开 CAS。
3. 选择 Codex 根目录，也就是包含 config.toml 的目录。
   如果设置过 CODEX_HOME，通常选择该目录；否则通常是用户目录下的 .codex。
4. CAS 会显示当前状态：官方 OpenAI 登录，或第三方 Responses API。

切换前请一定先退出 Codex，避免数据库或会话文件正在写入。


切换到第三方 API
----------------

1. 填写第三方 Base URL，例如：
   https://api.example.com

   CAS 会自动规范为：
   https://api.example.com/v1

2. 填写第三方模型名称。
3. 首次填写第三方 API Key。
4. 点击“切换到第三方 API”。
5. 切换完成后重新打开 Codex。

API Key 会使用系统安全存储：
- Windows：DPAPI
- macOS：Keychain
- Linux：Secret Service；没有密钥环时使用当前用户 0600 权限的本地加密保险库


切换回官方登录
--------------

1. 先彻底退出 Codex。
2. 填写官方模型名称。
3. 点击“切换到官方登录”。
4. 重新打开 Codex。

切回官方登录不会删除第三方 Key，也不会修改 auth.json。


中转站档案
----------

如果你有多个第三方中转站：

1. 填写 Base URL、模型和 API Key。
2. 点击“保存档案”。
3. 下次从“中转站档案”下拉框选择即可。
4. 删除档案会删除本机保存的对应加密 Key。


兼容模式
--------

如果第三方中转站报 Image generation 权限错误，可以勾选“兼容模式”后再切换到第三方 API。

兼容模式会临时关闭 Codex 生图、图片查看、Web 搜索和部分高级插件能力。切回官方或关闭兼容模式后，CAS 会恢复之前的配置。


修复会话列表
------------

如果 Codex 左侧历史会话为空：

1. 彻底退出 Codex。
2. 点击“修复会话列表”。
3. 重新打开 Codex 检查侧栏。

CAS 会先备份状态数据库，再恢复顶层用户会话的可见标记。不会修改会话正文。


恢复基础配置
------------

如果误删 model_provider、model，或者第三方 provider 配置损坏：

1. 点击“恢复基础配置”。
2. CAS 会备份 config.toml。
3. CAS 会恢复官方基础模型配置。

MCP、插件、沙箱、记忆、会话和 auth.json 会保留。


启动和关闭 Codex
----------------

- “启动 Codex”：打开 Codex 桌面应用或 codex 命令。
- “关闭 Codex 后台”：结束 Codex 相关后台进程。

切换 API 前建议先关闭 Codex 后台。


快捷键、鼠标侧键和开机启动
--------------------------

- “设置快捷键”：设置打开/隐藏 CAS 的全局快捷键。
- “鼠标侧键”：使用鼠标侧键打开/隐藏 CAS。
- “开机启动”：登录系统后 CAS 自动驻留后台。

Windows、macOS、Linux 都有单实例保护。重复打开 CAS 不会多开后台进程，而是唤醒已有窗口。


备份和安全
----------

CAS 会自动备份：
- config-switcher-backups：配置备份
- history_sync_backups：状态库和会话元数据备份

CAS 不会修改：
- 会话正文
- memories
- session_index.jsonl
- auth.json
- MCP 配置主体
- 插件配置主体

CAS 主要修改：
- config.toml 中的模型/provider 配置
- 状态数据库中的会话 provider 元数据
- 会话 JSONL 第一行的 provider 元数据
- 侧栏可见标记（仅修复会话列表时）


常见问题
--------

1. 为什么 Git 仓库里没有 .app？
   因为 .app 内自包含主程序超过 GitHub 普通 Git 单文件 100 MB 限制。请从 GitHub Releases 下载 zip。

2. 为什么 macOS 首次运行要允许？
   当前包是 ad-hoc 签名，不是 Apple Developer ID 公证包。macOS Gatekeeper 会要求手动确认一次。

3. 为什么切换后要重启 Codex？
   Codex 启动时读取配置。切换配置后必须重新打开 Codex 才会生效。

4. Base URL 要填到哪里？
   填站点根地址即可，例如 https://api.example.com。CAS 会自动补成 https://api.example.com/v1。

Codex API 切换器
================

文件：
跨平台版：
outputs\cross-platform\win-x64\CodexApiSwitcher.exe
outputs\cross-platform\osx-arm64\Codex API Switcher.app
outputs\cross-platform\osx-x64\Codex API Switcher.app
outputs\cross-platform\linux-x64\CodexApiSwitcher
outputs\cross-platform\linux-arm64\CodexApiSwitcher

兼容版：
outputs\CodexApiSwitcher.exe（原 Windows WinForms 版）

使用方法：
1. Windows 双击 win-x64 目录下的 CodexApiSwitcher.exe；macOS 打开对应架构的 .app；Linux 给 CodexApiSwitcher 加执行权限后运行。
2. 选择包含 config.toml 的 Codex 根目录。
   如果你设置了 CODEX_HOME，会优先使用该目录；否则选择自己的 Codex 根目录。
3. 第三方模式：
   - 填写 Base URL，例如 https://api.example.com
     工具会自动规范为 https://api.example.com/v1
   - 填写第三方模型名称
   - 首次填写 API Key
   - 点击“切换到第三方 API”
4. 官方模式：
   - 填写官方模型名称
   - 点击“切换到官方登录”
5. 切换前请彻底退出 Codex。切换完成后重新打开 Codex。
6. 跨平台版启用单实例保护；如果 CAS 已在托盘/菜单栏后台运行，再双击图标会打开已有窗口，不会多开后台进程。
6. 如果误删 model_provider、model 或损坏第三方 provider 配置：
   - 点击“一键恢复基础配置”
   - 工具会备份当前 config.toml
   - 恢复为官方 OpenAI 登录配置
   - MCP、插件、沙箱及其他无关配置保持不变

安全说明：
- 跨平台版为自包含单文件，不需要安装 Python、.NET 或 SQLite。
- SQLite 使用随应用打包的跨平台运行库。
- API Key 使用系统安全存储：Windows DPAPI、macOS Keychain、Linux Secret Service；
  Linux 无密钥环时使用当前用户 0600 权限的本地加密保险库。
- Linux 桌面仍需系统基础库，例如 glibc、libstdc++、ICU、fontconfig、X11/AppIndicator 相关库。
- macOS 首次使用全局快捷键/鼠标侧键可能需要在“系统设置 -> 隐私与安全性 -> 辅助功能”中授权。
- 每次切换前都会备份 config.toml，以及检测到的新旧状态数据库。
- 每次切换都会同步根目录旧库、sqlite 子目录活动库和会话 JSONL 第一行的 model_provider，
  防止 Codex 重启后把历史重新分回不同模式。
- 修改会话 JSONL 前会在 history_sync_backups 中逐文件备份。
- 工具只修改顶层用户会话的 provider 元数据和侧栏可见标记。
- 工具不修改会话正文、memories、
  memories_1.sqlite、session_index.jsonl 或 auth.json。
- 官方模式继续使用 Codex 已有的 OpenAI 登录状态。
- “恢复最近备份”可以回退最后一次配置修改。
- “一键恢复基础配置”只重建 model/model_provider，并清理损坏的 custom provider 段。
- 如果侧栏显示“暂无对话”，请先彻底退出 Codex，再点击“修复会话列表”。
  该操作会先自动识别并备份新旧状态数据库，然后只恢复顶层用户会话的可见标记。
